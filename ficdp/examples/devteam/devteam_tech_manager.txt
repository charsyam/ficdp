IoT를 위한 클라우드 환경의 필요성을 서술하시오

IoT 데이터 처리의 중요성

사물에 대한 데이터의 수집과 분석을 통해 가치있는 정보를 추출하기 위한 ‘데이터 처리 및 수집을 위한 인프라’가 필요함

빠르고 신속한 데이터의 분석과 예측 기능이 필요함

일반적인 서버-클라이언트 구조의 한계

짧은 시간내에 빅데이터가 생성되고, 이에 대한 신속한 저장 서비스가 필요함

서버 클라우드 구조에서의 고정된 자원 할당 구조이기에 데이터 규모에 따른 신속하고 유연한 자원 지원이 불가능함

따라서 실시간으로 수집/관리/분석 기능을 제공할 수 있는 인프라가 필요함

클라우드 환경의 데이터 처리 능력

데이터 규모에 따른 신속하고 유연한 자원의 제공을 통한 빅데이터 처리가 가능함

빅데이터의 실시간 수집/관리/분석이 가능함

빅데이터 처리를위한 인프라로 클라우드 환경이 적합함

클라우드 기반의 IoT가 상업적으로 출시됨

포그 컴퓨팅의 정의와 특징을 서술하시오

포그 컴퓨팅의 정의

기존 방식의 단점

기존 IoT에서 수집되는 모든 데이터는 정보의 중요도에 관계없이 일괄적으로 클라우드 시스템에 전송하여 처리됨(쓸모없는 데이터들도 모두 전송되늠 문제가 발생)

수집된 데이터가 모두 전송되기까지 소비되는 시간과 비용, 수집된 데이터를 관리하고 분석하는데 소비되는 시간과 비용이 큼

포그 컴퓨팅의 정의

클라우드 환경에서 제공되는 서비스의 특정 영역에 대한 역할을 책임짐 – 즉 처리할 수 있는 부분들에 대해서 처리를 하고 이를 클라우드로 보내지 않음.

네트워크 장비의 연산과 스토리지 자원으로 구성되는 서비스 인프라이며, 클라우드 처럼, 데이터 연산, 스토리지, 애플리케이션 서비스를 제공하지만, 중앙 서버를 거치지 않고 사용자에게 클라우드 처럼 느껴지는 서비스를 제공함.

포그 컴퓨팅의 특징

모든 데이터를 클라우드로 보내지 않음

실시간 처리가 필요한 데이터, 빠른 처리속도가 필요한 데이터를 선별하여 처리

데이터가 생성되는 네트워크 말단의 기기에서 직접 데이터의 연산/분석을 실행

이론적으로 접속횟수와 소비 대역폼을 절반으로 감소 시킴

데이터 이동에 따른 지연시간을 감소시키고, 각 지점마다 잠재된 장애 가능성을 해소할 수 있음
